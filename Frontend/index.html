OneBlock Plugin

A skyblock-style Minecraft experience where a single block is all you need. Break it, evolve it, survive on it.

Paper 1.21.x Spigot Purpur Open Source

OneBlock by WaldNetworks transforms a single block into a progressive challenge system. Each time you break the OneBlock, it respawns with different materials based on your current progression level — starting with humble stone and dirt, and gradually unlocking ores, mob drops, and rare resources as you advance.

It's a lightweight, server-side plugin designed for Paper, Spigot, and Purpur servers running Minecraft Java Edition 1.21.x.

Key Features

∞

Unlimited OneBlocks

Create multiple OneBlock instances across your server, each at any coordinates.

↑

Blocks evolve through fully customizable stages, with new resources at every level.

Installation

Get OneBlock running on your server in under two minutes.

Requirements

  • A Minecraft server running Paper, Spigot, or Purpur (1.21.x)
  • Java 21 or newer
  • Operator / console access to your server

Step-by-step

  1. Download the plugin
    Get OneBlock-1.1.3.jar from the Modrinth page.
  2. Place it in your plugins folder
    Copy the .jar file into your server's plugins/ directory. 
    your-server/
└── plugins/
    └── OneBlock-1.1.3.jar
  3. Restart your server
    Start or restart the server. OneBlock generates its default config.yml inside plugins/OneBlock/.
  4. Configure (optional)
    Edit plugins/OneBlock/config.yml to customise progression stages.
  5. Create your first OneBlock
    In-game (requires OneBlock.Admin permission), run: 
    /oneblock set <x> <y> <z>
Tip: After editing config.yml, restart the server for changes to take effect. Hot-reload is not currently supported.

Phases & Progression

Each break of the OneBlock advances progress. As levels rise, rarer and more valuable resources unlock automatically.

How It Works

The plugin tracks how many times each OneBlock has been broken. Once the break count reaches the level-size threshold defined for that stage in config.yml, the block advances to the next level and begins spawning items from the new stage's block pool.

The maximum level is set by Max_Level. Once reached, the block continues spawning from the final stage indefinitely.

Default Progression

1 Plains & Basics Stone, Dirt, Sand, Gravel, Oak Log 100

Tips

  • Create separate OneBlocks for different players or teams — each tracks independently.
  • Progress survives server restarts thanks to persistent storage.
  • Combine with a void world for the classic skyblock feel.

Commands

All OneBlock commands require the OneBlock.Admin permission. There are no player-facing commands — administration is done in-game by operators.

Command Reference

CommandDescription
/oneblock set <x> <y> <z> Creates a OneBlock at the specified world coordinates.
/oneblock delete <x> <y> <z> Removes the OneBlock at the given coordinates.
/oneblock deletebyid <ID> Removes a OneBlock by its unique numeric ID (shown in /oneblock list).
/oneblock deleteAllOneBlocks Removes every OneBlock on the server. Irreversible.
/oneblock list Lists all active OneBlocks with IDs, coordinates, and current level.

Permissions

NodeDescriptionDefault
OneBlock.Admin Grants access to all /oneblock commands OP only
Warning: /oneblock deleteAllOneBlocks permanently removes all OneBlocks and their progress. No confirmation prompt — use with caution.

Configuration

All progression stages are defined in plugins/OneBlock/config.yml. Customise block pools, level sizes, and the maximum level to suit your server.

File Location

your-server/
└── plugins/
    └── OneBlock/
        └── config.yml

Options Reference

KeyTypeDescription
Max_Level Integer Highest reachable level. Breaks beyond this use the final stage's pool.
Level_N Section Config block for level N (replace N with 1, 2, 3…).
Level_N.level-size Integer Breaks required to advance from level N to N+1.
Level_N.blocks List<String> Bukkit Material names (uppercase) that can spawn at this level. Chosen randomly per break.

Example config.yml

# Maximum level players can reach
Max_Level: 5

# ── Level 1: The Basics ──────────────────────
Level_1:
  level-size: 100
  blocks:
    - STONE
    - DIRT
    - SAND
    - GRAVEL
    - OAK_LOG
    - GRASS_BLOCK

# ── Level 2: Early Mining ────────────────────
Level_2:
  level-size: 200
  blocks:
    - COBBLESTONE
    - OAK_PLANKS
    - IRON_ORE
    - COAL_ORE
    - LAPIS_ORE
    - COPPER_ORE

# ── Level 3: Underground ─────────────────────
Level_3:
  level-size: 300
  blocks:
    - GOLD_ORE
    - REDSTONE_ORE
    - ANDESITE
    - DIORITE
    - GRANITE
    - MOSSY_COBBLESTONE

# ── Level 4: Nether ──────────────────────────
Level_4:
  level-size: 400
  blocks:
    - NETHERRACK
    - NETHER_BRICKS
    - SOUL_SAND
    - MAGMA_BLOCK
    - CRIMSON_STEM
    - WARPED_STEM

# ── Level 5: End Game ────────────────────────
Level_5:
  level-size: 500
  blocks:
    - DIAMOND_ORE
    - EMERALD_ORE
    - ANCIENT_DEBRIS
    - END_STONE
    - OBSIDIAN
    - CRYING_OBSIDIAN
Block names must match the Bukkit Material enum exactly (uppercase, underscores). Full list in the Bukkit JavaDocs ↗.
